# create-skill

새로운 커스텀 슬래시 커맨드(스킬)를 `.claude/commands/` 아래에 생성한다.

## 사용법

```
/create-skill <skill-name> [설명]
```

- `skill-name`: 생성할 스킬 파일명 (kebab-case). 이 이름이 곧 `/skill-name` 커맨드가 된다.
- `설명` (선택): 스킬이 해야 할 일을 한 줄로 설명. 생략하면 대화로 파악한다.

## 절차

**Step 1 — 목적 파악**

`$ARGUMENTS`에서 스킬 이름과 설명을 추출한다. 설명이 없으면 "이 스킬이 해야 할 일을 설명해주세요"라고 묻는다. 충분히 파악됐으면 바로 진행한다.

**Step 2 — 저장 위치 결정**

- 프로젝트 전용 스킬: `.claude/commands/<name>.md` (현재 워킹 디렉토리 기준)
- 사용자가 "글로벌" 또는 "전역"으로 저장하길 원하면: `~/.claude/commands/<name>.md`
- 기본값은 프로젝트 로컬.

**Step 3 — 스킬 파일 작성**

아래 구조로 `.claude/commands/<name>.md`를 작성한다:

```markdown
# <skill-name>

<스킬이 하는 일 한 줄 요약>

## 사용법
/skill-name [인자]

## 절차
1. ...
2. ...

## 출력
- ...
```

작성 기준:
- 스킬 파일은 **Claude에게 주는 지시문**이다. "너는 X를 해야 한다"는 형식으로 작성.
- `$ARGUMENTS` 플레이스홀더를 사용해 호출 시 넘어온 인자를 참조할 수 있다.
- 단계별 절차를 명확히 나열해 재현 가능하게 만든다.
- 외부 도구 사용(Bash, Read, Write 등)이 필요하면 명시한다.
- 스킬의 결과물(파일 생성, 브라우저 열기 등)을 **출력** 섹션에 명시한다.

**Step 4 — 완료 보고**

생성된 파일 경로와 호출 방법을 사용자에게 알린다:

```
✓ .claude/commands/<name>.md 생성 완료
호출: /<name> [인자]
```

## 주의사항

- 이미 같은 이름의 파일이 있으면 덮어쓰기 전에 사용자에게 확인한다.
- 스킬 이름은 kebab-case를 사용한다 (예: `create-skill`, `mermaid-diagram`).
- 스킬 파일 안에 비밀 키, 토큰 등 민감한 정보를 넣지 않는다.
