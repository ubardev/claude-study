# 태그 기능 정의서 (확정)

## 기능 개요

노트에 태그를 추가하고 관리할 수 있다.

## 기능 요구사항

- 노트에 태그를 추가할 수 있다.
- 노트에 추가된 태그를 삭제할 수 있다.
- 태그 목록을 노트 상세 화면에서 확인할 수 있다.

---

## 데이터 구조

### Note 타입 확장

```ts
interface Note {
  id: string;
  title: string;
  content: string;
  tags: string[]; // 추가 (기존 미구현 필드)
  createdAt: string;
  updatedAt: string;
}
```

- 태그는 문자열 배열로 저장한다.
- 태그가 없는 기존 노트는 빈 배열 `[]`로 취급한다.
- 중복 태그는 허용하지 않는다 (대소문자 구분, 정확히 일치 시 중복 판정).

### JSON Server

- `db.json`의 각 note 객체에 `"tags": []` 필드를 추가한다.

---

## 저장 방식

- 태그 추가·삭제 즉시 `PATCH /notes/:id` API를 호출해 저장한다.
- 노트 저장 버튼(제목/본문 저장)과 별개로 동작한다.

---

## UI 패턴

### 태그 입력

- `NoteEditor` 내 태그 입력창(`<input>`)을 노트 본문 아래에 배치한다.
- 입력창에 태그명을 입력하고 **Enter 키**를 누르면 태그가 추가된다.
- 추가 성공 후 입력창은 비워진다.

### 태그 표시

- 추가된 태그는 입력창 위(또는 근처)에 뱃지(pill) 형태로 나열한다.
- 각 뱃지에는 삭제 버튼(`×`)이 있으며 클릭 시 즉시 삭제된다.

---

## 유효성 검사

| 규칙                | 처리                   |
| ------------------- | ---------------------- |
| 입력값 앞뒤 공백    | 자동 trim              |
| trim 후 빈 문자열   | 추가 불가 (입력 무시)  |
| 중복 태그           | 추가 불가 (입력 무시)  |
| 최대 글자 수        | 20자 초과 시 추가 불가 |
| 허용 문자           | 제한 없음              |
| 노트당 최대 태그 수 | 제한 없음              |

---

## API

기존 `src/api/notes.ts`에 태그 전용 함수를 추가하지 않는다.  
태그 변경은 기존 `updateNote(id, { tags: [...] })` 호출로 처리한다.

---

## 컴포넌트

- 태그 UI는 별도 컴포넌트 `TagEditor`로 분리한다.
- `TagEditor`는 `tags: string[]`, `onAdd: (tag: string) => void`, `onDelete: (tag: string) => void` props를 받는다.
- API 호출(즉시 저장)은 `NoteEditor`에서 담당하고, `TagEditor`는 순수 UI만 처리한다.

## 엣지케이스

| 케이스                               | 처리                                                             |
| ------------------------------------ | ---------------------------------------------------------------- |
| 태그 추가 중 API 실패                | `console.error()` 출력, UI는 낙관적 업데이트 없이 원래 상태 유지 |
| 태그가 없는 기존 노트                | `tags` 필드 없으면 `[]`로 폴백                                   |
